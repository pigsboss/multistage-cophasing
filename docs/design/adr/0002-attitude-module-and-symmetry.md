# ADR-0002: Attitude 模块与星历‑姿态架构对称性设计

## 状态

已接受

## 背景

`mission_sim/core/spacetime` 当前仅包含星历（`ephemeris`）子包，负责天体质心位置与速度的获取。天体自转轴变化（岁差、章动、极移等）尚无统一建模。

为了支持 LEO 高能天文卫星的本底建模、地磁场配准以及地月空间任务仿真，需要：
- 引入天体姿态计算能力；
- 提供从惯性天球坐标系到天体固连系的变换；
- 保持与星历模块相同的架构分层：解析即时模型、SPICE 高精度模型、表格插值模型。

同时，现有星历侧只能通过预先生成的表格（`Ephemeris`）查询状态，缺少“透传”式即时计算模型，导致星历与姿态两侧不对称，增加了使用成本。

## 决策

### 1. 新增 `attitude` 子包

在 `mission_sim/core/spacetime/` 创建 `attitude/` 子包，与 `ephemeris/` 并列。其职责为直接提供天体姿态的旋转矩阵/四元数，不依赖预生成表格。
