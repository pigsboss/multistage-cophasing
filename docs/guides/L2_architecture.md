# MCPC 框架 L2 级（协同效能）架构设计与开发总纲 V2.0

**文档状态**：最终版 (Final)
**目标层级**：L2 (The Mission)
**核心主旨**：面向任务规划与协同效能评估，在宏观尺度上验证多星编队的协同可行性、控制代价与任务效率。

---

## 🏛️ 一、 工程定位与核心边界

### 1. 物理视角的跃迁
从 L1 级的单星绝对轨道演化，跃迁为以编队虚拟质心或主星为参考的**多质点相对动力学模型**。焦点从“单星星历基准”转移至“多星构型演化”。

### 2. 严守“姿态锁定”假设与解耦边界
为避免过早陷入 L4 级的 6-DOF 强耦合泥潭，L2 级**显式声明姿态动力学与平动动力学完全解耦**：
* 推力矢量假设始终与相对坐标系（如 LVLH）中的期望方向完美一致，无指向误差。
* 星间链路（ISL）测量假设在相对坐标系中直接获得相对位置/速度，无姿态测量噪声与视场遮挡。
* **架构契约**：此“姿态锁定”假设的失效边界与系统间强耦合效应由未来的 L3/L4 级负责处理。L2 级坚决不承担姿态耦合与内部机构微振动分析。

### 2. 区分“近地”与“深空”的相对动力学
针对不同物理环境，定义统一的 `RelativeDynamics` 抽象基类，并提供针对性实现：
| 仿真场景 | 相对动力学模型 | 坐标系契约 |
| :--- | :--- | :--- |
| **LEO / GEO (二体)** | Hill-Clohessy-Wiltshire (C-W) 方程 | LVLH (主星轨道坐标系) |

## 📡 三、 测控与执行域 (Perception & Actuation) 架构设计

### 1. ISL 星间链路：截断统计误差模型
抛弃理想测距假设，构建符合真实工程限制的 `ISL_Network`：
* **建链物理约束**：当相对距离 > `max_link_distance` 或相对速度 > `max_link_velocity` 时，测量值为 `NaN`（模拟链路中断）。
* **测量误差合成**：
  1. **高频白噪声**：服从高斯分布的短期观测跳动。
  2. **低频漂移/底噪**：采用一阶马尔可夫或随机游走模型，模拟不可消除的基线绝对不确定度（防止“积分无限长则误差为零”的谬误）。

### 2. 推力器：宏观约束的降维等效
将复杂的硬件特征（如冷气、电推的延迟与死区）降维为四个宏观参数传入 `ThrusterModel`，避免在 L2 级进行高频积分：
```python
class ThrusterModel(ABC):
    def __init__(self, max_force, min_impulse_bit, force_noise_std, control_cycle_s):
        self.max_force = max_force               # 最大推力限制 (N)
        self.min_impulse_bit = min_impulse_bit   # 最小冲量位 (N·s) - 决定稳态精度下限
        self.force_noise_std = force_noise_std   # 推力输出白噪声比例
        self.control_cycle_s = control_cycle_s   # 控制周期 (离散化，等效模拟时间响应滞后)
```

## 🧠 四、 控制律与状态机 (GNC Logic)

在 `FormationController` 中引入**三段式状态机**，实现全寿命周期的编队自治管理：

| 编队状态 | 切入条件 (Entry) | 退出/切换条件 (Exit) |
| :--- | :--- | :--- |
| **1. 编队生成** | 初始入轨完成，通过地基测控抵近 | 相对位置误差 < 阈值，且相对速度 < 阈值 |
| **2. 编队保持** | 误差满足观测要求 | 接收到地面/时序的“重构”指令 |
| **3. 编队重构** | 接收到重构指令 | 新构型误差 < 观测阈值，且 **稳定时间 > 冷却时间** |

## 📊 五、 核心效能产出 (Efficacy Outputs)

L2 级的直接产出必须回答任务系统工程最关心的三个问题：

### 1. 任务时间效率（观测占空比）
* **冷却整定时间**：定义为“从重构指令结束”到“所有基线误差均收敛并稳定在观测阈值内”所耗费的时间间隔。
* **占空比指标**：输出单次任务周期内，**编队稳定可用时间 / 任务总时间** 的比例，直接支撑科学载荷的工作调度。

### 2. 宏观光程差（OPD）差分评估
提供清晰的干涉基线接口 `InterferometerBaseline`，直接输出 $\Delta \vec{\rho}$ 包络：
* 提取“集光器 1 - 合束器”与“集光器 2 - 合束器”的两条基线向量。
* 输出其在测控误差与推力白噪声作用下的**相对变化差值与速度差**。
* 评估该宏观差分包络是否为未来的微观 OPD 光学补偿装置留出了足够的动态余量。

### 3. 常规编队物理账单
* 输出构型初始化的包络图、以虚拟质心为原点的 3D 相对轨迹安全边界。
* 输出阵列协同总 $\Delta V$ 开销（燃料预算本底）。

## 📋 六、 L2 级 Sprint 开发路线图 (WBS)

* **Sprint 1: 基础设施与相对坐标系基建**
  * 扩展 `CoordinateFrame`，增加 `LVLH`。
  * 实现 ECI/Rotating 到相对标架的双向严密转换。
  * 定义 `RelativeDynamics` 抽象接口与多星嵌套 `FormationState` 容器。
* **Sprint 2: 编队动力学积分工厂**
  * 实现基于组合模式的 `FormationSimulation` 架构。
  * 引入器间互引力模型，验证无控状态下 $N$ 星漂移轨迹与经典理论的对齐度。
* **Sprint 3: 统计测控与宏观推力映射**
  * 落实 `ISL_Network` 的 `NaN` 截断逻辑与低频底噪。
  * 落实 `ThrusterModel` 的 `min_impulse_bit` 限幅与 `control_cycle_s` 离散化。
* **Sprint 4: GNC 三段式状态机**
  * 编写涵盖“生成-保持-重构”的切换逻辑与收敛阈值判定。
  * 开发基于相对状态的平动控制器（LQR/MPC）。
* **Sprint 5: 效能分析与可视化**
  * 编写 `OPD_Analyzer` 统计基线差分向量。
  * 编写“冷却时间”统计算法与观测占空比报告生成器。
  * 升级 `visualizer_L2.py`，绘制相对包络图与重构时序图。
