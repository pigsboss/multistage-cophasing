# MCPC L1 级架构设计文档：基准与定标 (The Baseline)

## 一、 架构定位与核心目标

L1 级架构作为 MCPC 框架的通用数字底座，面向工程总体层面，定性且准确定量地解决航天器“飞什么轨道、基本环境如何、与地面测控接口是什么”的问题。

其核心产出必须包含：

1. **全域标称轨道（Nominal Orbit）的生成**：摆脱单一平衡点限制，支持从近地开普勒轨道到深空非线性流形轨道的按需生成。
2. **基准燃料账单**：计算克服当前轨道主导摄动（如 LEO 的大气/J2，或深空的光压）进行轨道维持（Station Keeping）的基准 $\Delta V$ 消耗。

## 二、 核心模块划分与演进设计

我们将 L1 级架构拆解为以下四个核心服务，并确立“环境与轨道可插拔”的设计原则：

### 1. 动力学底座层 (The Plant)

- **设计约束**：宏观轨道质点，仅加载与当前轨道区域匹配的主导摄动，无姿态。
- **架构升级策略（可插拔环境引擎）**：
  - **重构 `CelestialEnvironment` 为力学注册表（Force Registry）**：根据任务配置动态加载引力与环境模型。
    - **LEO/GEO/HEO**：加载高阶非球形地球引力（如 $J_2 \sim J_4$）及大气阻力模型。
    - **地月空间 (Cislunar)**：加载地月双星高精度星历引力。
    - **深空/平动点/Earth-trailing**：加载多体引力及参数化面积质量比（A/m）的球对称太阳光压模型（Cannonball SRP）。

### 2. 控制与算法层 (Logic) —— [本次核心修正点]

- **设计约束**：绝对轨道维持，动态星历追踪 (Station Keeping)。
- **架构升级策略（多态标称轨道生成工厂）**：
  - **重构 `TrajectoryGenerator` 模块**：在仿真预处理阶段，提供多种底层算法生成标称星历（$X_{nominal}(t)$）：
    - **开普勒/解析解生成器**：针对 LEO/GEO，基于轨道根数（含 $J_2$ 长期项修正）生成参考星历。
    - **微分修正生成器 (Differential Correction)**：针对日地/地月平动点任务，求解流形附近的闭合 Halo、Lissajous 或 NRHO（近直线晕轨道）。
    - **数值积分/高精度历表生成器**：针对 Earth-trailing / Earth-leading 轨道（类似 Spitzer 或 Kepler 望远镜），通过输入初始状态与太阳辐射压系数，在真实行星历表（如 JPL DE440）下长期积分生成参考轨迹。
  - **闭环追踪逻辑**：GNC 误差计算更新为 $e(t) = X_{nav}(t) - X_{nominal}(t)$，确保单星能够在其特定的“本征轨道”上平稳巡航。

### 3. 测量与信息层 (Perception)

- **设计约束**：地基测控，绝对遥测，粗量级位置/速度更新。
- **架构升级策略（测控弧段模拟）**：
  - 引入**离散时间与可视窗口（Visibility Window）模型**：根据轨道类型（如深空盲区或 LEO 测站覆盖范围），设定测控弧段与采样率。航天器在无测控弧段时，GNC 强制依赖星上轨道外推（Propagator）。

### 4. 执行层 (Actuation)

- **设计约束**：理想推力中心，无硬件约束的 $\Delta V$ 输出。
- **架构升级策略（精确燃料本底核算）**：
  - 在 `SpacecraftPointMass` 中增加连续积分的 $\Delta V$ 统计器。为工程总体提供评估报告（例如：Earth-trailing 轨道的年均维持 $\Delta V$ 是否显著低于 Halo 轨道）。

## 三、 L1 级核心数据流向 (Data Flow)

整个 L1 的生命周期被严格划分为以下边界，预处理阶段被赋予了全域多态能力：

1. **[预处理阶段：全域星历生成]**

   读取任务配置（区域+轨道类型） -> 触发 `TrajectoryGenerator` 工厂（开普勒解析 / 微分修正 / 历表外推） -> 生成该任务的全局标称轨道离散星历表（Ephemeris Dict）。

2. **[感知阶段：真实受限测控]** 物理真值 -> 检查测控可视弧段 -> 叠加深空/近地定轨噪声 -> `GroundStation` 下发 -> 导航状态更新。

3. **[决策阶段：泛化追踪]** 当前时间 $t$ -> 从预处理星历表中插值获取 $X_{nominal}(t)$ -> 计算追踪误差 -> LQR / 轨道控制律解算 -> `force_cmd`。

4. **[物理阶段：匹配环境]** 动力学引擎（动态加载的主导摄动力 + `force_cmd`） -> 数值积分 -> 更新真值。

5. **[后处理阶段：定标报告]** 产出特定轨道类型下的 Station Keeping 燃料预算底账与 3D 标称偏离图。
